You are working in the `maniator/blipit-legends` repository. Before starting League Mode v2, create a complete BMAD implementation plan for the QA follow-up work below.

- Identify which BMAD agents/skills should own each slice.

## Goal

Finish League Mode v1 and related pre-v2 QA polish so the app is safe to expand into League Mode v2.

## Critical Issues to Plan

### 1. Mixed-mode managed team selection

Current issue:

- In Mixed mode, the Review & Create step has a “Which team will you manage?” field.
- The field is unreadable/low-contrast and does not work reliably as a dropdown.
- The user can create the season without explicitly choosing a managed team.
- The app appears to auto-assign a managed team.

Plan requirements:

- Make the control readable and accessible.
- Use a native select unless there is a strong reason not to.
- Ensure mouse and keyboard interaction works.
- Require explicit managed-team selection before season creation.
- Show inline validation when missing.
- Store the selected managed team correctly.
- Add unit/component/e2e coverage.

Suggested BMAD skills/agents:

- UX/frontend forms agent.
- A11y agent.
- QA/e2e agent.

### 2. Watch mode must not expose manager controls

Current issue:

- Launching a league game via **Watch** still exposes Manager Mode and manager controls.
- Spectators can access substitution and decision tuning.

Plan requirements:

- Add an explicit game launch mode/state, such as `watch`, `manage`, `auto`.
- Preserve the mode through routing/query params or game context.
- Hide/disable manager controls in watch mode.
- Ensure manager controls remain available in play/manage mode.
- Add e2e coverage for both paths.

Suggested BMAD skills/agents:

- Game state/routing agent.
- Frontend gameplay UI agent.
- QA/e2e agent.

### 3. Headless auto-simulation for managed-team games

Current issue:

- On season home, **Advance Season** stops at “Your next game is ready!” and offers only **Play in Manager Mode** or **Watch**.
- There is no way to auto-simulate the managed-team game without watching or playing.

Plan requirements:

- Add “Auto-simulate game” or equivalent CTA.
- Run the scheduled game headlessly with AI managing both teams.
- Write final score to season game record.
- Update standings and current day progression.
- Handle errors and idempotency.
- Add tests for score write-back and standings update.

Suggested BMAD skills/agents:

- League simulation/storage agent.
- Backend testing agent.
- Frontend season UX agent.
- QA/e2e agent.

### 4. Wizard validation

Current issue:

- Wizard can progress with invalid team selection.
- Errors appear late on review instead of inline.

Plan requirements:

- Validate per step before enabling Next.
- Handpick mode must enforce exactly 8 teams for Mini.
- Mixed mode must show picked/autogenerated counts and require managed team.
- Errors must be visible near relevant fields.
- Preserve wizard state in sessionStorage if that is part of v1 spec.

Suggested BMAD skills/agents:

- Frontend wizard forms agent.
- UX/a11y agent.
- Unit/e2e test agent.

### 5. SeasonTeamPage and roster/fatigue view

Current issue:

- SeasonTeamPage appears missing or not connected.
- Users cannot inspect season roster/fatigue in context.

Plan requirements:

- Implement `/leagues/:seasonId/teams/:seasonTeamId`.
- Show roster snapshot, lineup, bench, pitchers, and fatigue/status pills.
- Use v1 shared components such as `StatusPill`, `StatusBanner`, `EmptyState`, `ModalShell` where appropriate.
- Add navigation from season home/schedule/hub where appropriate.

Suggested BMAD skills/agents:

- League UI agent.
- Component library agent.
- A11y agent.
- QA/e2e agent.

### 6. Roster-edit lock

Current issue:

- Teams in active seasons do not clearly show locked state in Manage Teams.
- Editing/destructive actions appear available from global team management.

Plan requirements:

- Surface locked state for teams in active seasons.
- Disable or hide destructive actions for locked teams.
- Ensure storage guard prevents unsafe edits even if UI is bypassed.
- Add tests for UI lock and storage enforcement.

Suggested BMAD skills/agents:

- Storage/domain integrity agent.
- Custom team editor agent.
- QA/e2e agent.

### 7. League hub canonical states

Current issue:

- `/leagues` does not fully behave as canonical league hub.

Plan requirements:

- Empty state with Start Season CTA.
- Active season continue state.
- Completed/champion state.
- Start-new-season flow with clear one-active-season constraints.
- Use shared components for banners/empty states.

Suggested BMAD skills/agents:

- Routing/page-state agent.
- UX agent.
- QA/e2e agent.

- Watch mode cannot enable manager controls.
- Play/manager mode still allows intended management controls.
- User can auto-simulate a managed-team game without watching/playing.
- Headless simulated games write scores and update standings.
- Schedule reflects completed and upcoming games accurately.
- SeasonTeamPage exists and is reachable.
- SeasonTeamPage shows roster/fatigue/status info.
- Active-season teams are protected from unsafe edits.
- Manage Teams player reordering works and persists.
- Team export/import gives clear feedback and supports files.
- Saves export/import gives clear feedback and round-trips.
- Key icon controls have accessible names.
- Keyboard navigation and focus states are usable.
- Mobile layouts are tested directly.
