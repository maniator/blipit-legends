# League Mode v1 QA follow-up before v2: Mixed mode, watch/manage permissions, headless sim, and UX/a11y cleanup

## Context

This issue captures the QA follow-up work that should be planned and completed before starting League Mode v2. League Mode v1 is partially functional, but several flows do not yet match the intended v1 behavior from `_bmad-output`.

The most important issues are:

- Mixed-mode season setup does not let the user reliably choose which team to manage.
- The mixed-mode managed-team dropdown is unreadable/inaccessible.
- Watch mode still exposes manager controls.
- There is no clear way to auto-simulate/headlessly advance a managed-team game without watching or playing.
- Season-team pages and roster/fatigue views appear incomplete or missing.
- Several non-league UX/a11y issues should be fixed before v2 expands the app surface.

## P0 Issues

### 1. Mixed-mode “Which team will you manage?” dropdown is unreadable and inaccessible

Observed:

- The field appears tiny/low contrast.
- Clicking the field does not reliably open options.
- Keyboard interaction is unreliable.
- The season can be created even when the user does not explicitly choose the managed team.
- The app appears to auto-assign one of the hand-picked teams.

Expected:

- The user must explicitly select which hand-picked team they manage.
- The control must be legible.
- It must work with mouse and keyboard.
- Creation should be blocked if no managed team is selected.

### 2. Watch mode allows manager controls

Observed:

- Opening a scheduled league game via **Watch** still exposes the **Manager Mode** toggle.
- The user can access substitution and decision tuning controls.

Expected:

- Watch mode is spectator-only.
- Manager Mode toggle and manager-only controls are hidden/disabled.
- The game route/state preserves whether the game was launched as watch or manage.

### 3. Missing headless simulation for managed-team game

Observed:

- On season home, **Advance Season** reaches a “Your next game is ready!” state.
- Options are **Play in Manager Mode** or **Watch**.
- There is no option to auto-simulate that game without watching/playing.

Expected:

- User can auto-simulate a managed-team game.
- AI manages both teams.
- Final score is written back to schedule.
- Standings update correctly.
- Season advances when the day is complete.

### 4. Wizard validation permits invalid progression

Observed:

- Team selection can proceed before the current step is valid.
- Errors appear late on review rather than inline on the relevant step.

Expected:

- Current step validates before **Next** is enabled.
- Handpick mode enforces exactly 8 teams for Mini.
- Mixed mode clearly shows picked vs autogenerated count.
- Mixed mode requires a managed team selection.

## P1 Issues

### 5. SeasonTeamPage appears missing/unused

Expected route/behavior:

- `/leagues/:seasonId/teams/:seasonTeamId`
- Shows roster snapshot, lineup/bench/pitchers, fatigue/status pills.
- Read-only when appropriate.

### 6. Roster-edit lock is not obvious in Manage Teams

Expected:

- Teams in active seasons show a lock/banner.
- Destructive actions are hidden/blocked.
- Storage guard still enforces lock.

### 7. League hub canonical states are incomplete

Expected:

- Empty state.
- Active season continue state.
- Completed/champion state.
- Start-new-season path.
- Clear handling of one-active-season constraint.

### 8. Season home next-game UX is incomplete

Expected:

- Show next game/day preview.
- Offer Play, Watch, and Auto-simulate when the managed team is involved.
- Allow headless advance for non-managed games.

## Non-league QA items to include in the implementation plan

### Manage Teams

- Drag-and-drop/reordering players did not work reliably.
- Need persistent batting order and pitching rotation reordering.
- Need keyboard-accessible reordering alternative.

### Team export/import

- Export has no visible feedback and does not clearly download a file.
- Import has no clear success/failure feedback.
- Need JSON file download/upload flow plus validation messages.

### Saves modal

- Save export/import lacks feedback.
- Need save export file download, import validation, and round-trip tests.

### Audio control

- Music/audio icon lacks clear label/tooltip/volume control.
- Add accessible label and persisted audio preference.

### PWA/offline

- Install affordance exists, but offline capability is unclear.
- Add offline banner/fallback if PWA support is intended.

### General readability/accessibility

- Form controls, dropdowns, prompts, and small labels need contrast/font-size audit.
- Add focus states and ARIA labels to icon-only controls.
- Ensure keyboard operation for sliders, dropdowns, and game controls.

## Suggested BMAD / implementation order

1. Mixed-mode managed team selection and validation.
2. Game launch mode state: watch vs manage vs auto-sim.
3. Watch-mode permission enforcement.
4. Headless managed-team game simulation.
5. Season schedule/standings write-back tests.
6. SeasonTeamPage and roster/fatigue UI.
7. Roster-edit lock UI and storage guard tests.
8. League hub state completion.
9. Manage Teams reordering and import/export improvements.
10. Saves modal import/export feedback.
11. Accessibility/readability pass.
12. Mobile/responsive QA.
