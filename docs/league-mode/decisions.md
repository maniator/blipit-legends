# League Mode — Locked Decisions

> Companion to [`README.md`](README.md) and [`roadmap.md`](roadmap.md). Every row is a design question we answered during planning. If a design question is not listed here, it has **not** been decided yet — bring it to the next planning session.

## Setup & scope

| #   | Topic                                | Decision                                                                                                                                                                                                                                                                                                                                        | Notes                                                                                                                                                                                             |
| --- | ------------------------------------ | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 1   | League sizes                         | **Presets only**: Mini (8) / Standard (16) / Full (24). No custom counts in any version.                                                                                                                                                                                                                                                        | v1 = Mini · v2 = +Standard · v4 = +Full.                                                                                                                                                          |
| 2   | Season length                        | **Presets only**: Sprint (30) / Standard (60) / Marathon (120).                                                                                                                                                                                                                                                                                 | v1 = Sprint · v2 = +Standard · v4 = +Marathon.                                                                                                                                                    |
| 3   | Offseason in v1?                     | **Out of v1.** Optional minimal carryover (heal IL + reset fatigue + keep rosters + keep league membership) ships in v4 as an opt-in toggle on "Start New Season".                                                                                                                                                                              | v1 simply ends the season → declares champion → "Start New Season" wizard.                                                                                                                        |
| 4   | Manager Mode scope                   | **User team only** for v1–v3. v4 adds optional "manage all teams" toggle in setup.                                                                                                                                                                                                                                                              | Mirrors exhibition mental model; halves the v1 UI surface.                                                                                                                                        |
| 5   | DH                                   | **Per-league radio at setup.** Quick presets: "Classic AL/NL" (League A on, League B off), "Universal DH" (both on), "Pitchers Hit" (both off). Custom is allowed.                                                                                                                                                                              | Interleague games (v2+) use **home team's rule**.                                                                                                                                                 |
| 6   | Fatigue in v1                        | **Yes — pitcher arm fatigue ships in v1.** Position-player wear and injuries wait for v2.                                                                                                                                                                                                                                                       | Pitcher fatigue is the lightest realism slice (just rest-day tracking + ineligibility-to-start).                                                                                                  |
| 7   | Custom + built-in teams in a league  | **Same data type — one library.** Built-in, user-created, and autogenerated teams all live in the existing `customTeams` collection. **No new "league teams" collection.**                                                                                                                                                                      | League state references teams by `customTeamId`. Autogenerated teams are written into `customTeams` at setup time so they show up in the user's library and can be reused, exported, edited, etc. |
| 7b  | Roster-edit lock for in-season teams | **A `customTeams` doc that is in any league with an `active` season is read-only in the Custom Team Editor.** Edits are only permitted via the season's sanctioned mutation paths (trades, IL moves, call-ups, lineup/rotation changes inside the season UI). Lock releases the moment the season's `status` becomes `complete` or `abandoned`. | Enforced at the storage layer (write guard) so route-level UX cannot be bypassed. The Custom Team Editor route still renders for read access and surfaces a banner explaining the lock.           |
| 8   | Concurrent seasons                   | **One active season at a time.** Setup wizard refuses to start a new one until current is `complete` or explicitly abandoned.                                                                                                                                                                                                                   | Historical completed seasons remain queryable.                                                                                                                                                    |

## Realism knobs

| #   | Topic                       | Decision                                                                                                                                                                                                                                                                                                                              | Notes                                                                                                                                                                                                                                 |
| --- | --------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 9   | Injury rate                 | **1.5% per active-lineup player-game** in regular season. Pinned to `rulesetVersion` — loading a season created under an older ruleset preserves that ruleset's value.                                                                                                                                                                | See [`fatigue-and-injuries.md`](fatigue-and-injuries.md) for the authoritative constant. Scales to ×0.5 in playoffs (#19). Only regular-season active-lineup players roll; bench players do not.                                      |
| 10  | Pitcher fatigue UI          | **Pill badge on player rows.** States: `🟢 Fresh` / `🟡 Tired` / `🔴 Spent`. Shown in lineup card, season team page, manager-mode prompts. **v1 ships with a hover/tap tooltip** surfacing "X days rest, availability Y%".                                                                                                            | Tooltip reuses the existing title/aria-describedby pattern; see `style-guide-additions.md` for the `<StatusPill>` variant spec.                                                                                                       |
| 11  | Fatigue/wear knob magnitude | **Trust the design defaults.** Pitchers: up to ~12% effect (meaningful for late-game decisions). Position players: up to ~5% effect (subtle, hint-driven).                                                                                                                                                                            | Post-v1 / post-v2 tuning passes run via `@baseball-manager` on real season logs and are tracked as acceptance items in `agent-prompts/v2.md` and `agent-prompts/v3.md`; re-tuning is a `rulesetVersion` bump.                         |
| 12  | AI rest behavior            | **Role-dependent.** Pitchers: strict eligibility threshold (SP ≥ 0.70, RP ≥ 0.35). Position players (v2+): **soft probability** `restProb = clamp((wear − 6)/4, 0, 1)` rolled after lineup, before box-score sim, in fixed iteration order (lineup positions 1–9 home, then 1–9 away — matches the injury-roll iteration convention). | Soft probability avoids abrupt "fine today, benched tomorrow" behavior. See [`fatigue-and-injuries.md`](fatigue-and-injuries.md).                                                                                                     |
| 13  | League-play roster minimums | **v1:** lineup 9 + bench 3 + 5 SP + 3 RP = 20 active. **v2+:** lineup 9 + bench 5 + 5 SP + 4 RP = 23 active.                                                                                                                                                                                                                          | Approved by `@baseball-manager`: 3 RP is the v1 floor for a credible bullpen; 4 RP in v2+ keeps the bullpen viable once full fatigue + injury systems are live. Setup wizard offers "auto-fill from autogen" for teams below minimum. |

## Trades & playoffs

| #   | Topic                        | Decision                                                                                                                                                                                                                                                                                                                                                                                                                 | Notes                                                                                                                                                                                                                                            |
| --- | ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 14  | Trade deadline               | **Default = 70% of the season.** Configurable at setup: slider 50%–100% or "no deadline".                                                                                                                                                                                                                                                                                                                                | Past the deadline, all trade UIs are read-only.                                                                                                                                                                                                  |
| 15  | AI trade aggressiveness      | **Middle / "moderate"** = 2–3 trades per team per season. Configurable: passive / **moderate** / active.                                                                                                                                                                                                                                                                                                                 |                                                                                                                                                                                                                                                  |
| 16  | User veto on AI ↔ AI trades  | **No veto.** Trades land in transactions feed silently.                                                                                                                                                                                                                                                                                                                                                                  | Realistic + low-friction. User can still scrutinize transactions.                                                                                                                                                                                |
| 17  | Playoff format               | **4 teams per league.** WC bo3 / LCS bo5 / BCS bo7 (default). Presets "Short" (1/3/5) and "Long" (5/7/7) selectable at setup.                                                                                                                                                                                                                                                                                            | Reseed after each round by regular-season record.                                                                                                                                                                                                |
| 18  | Tiebreaker                   | **Ordered chain (iterative for 3-way+):** (1) head-to-head record, (2) intra-division record (v2+ only, when divisions exist), (3) run differential vs the tied group, (4) full-season run differential, (5) coin flip via seeded PRNG using a per-tie-key sub-stream (`mulberry32(fnv1a(\`${masterSeed}:tiebreak:${sortedTiedIds}\`))`). Three-way+ ties resolve highest-seed-first then recurse on the residual group. | Matches MLB-style ordering; the seeded coin flip preserves replay determinism. v1 only ever needs steps 1, 4, and 5 (no divisions, no playoff seeding). Full chain operationalized in [`playoffs-and-trades.md`](playoffs-and-trades.md) for v3. |
| 19  | Playoff fatigue/injury rules | **Full pitcher fatigue + ×0.5 injury rate.** Position-player wear unchanged.                                                                                                                                                                                                                                                                                                                                             | "Big games, fewer freak injuries but October tempo still matters." Approved by `@baseball-manager`.                                                                                                                                              |
| 20  | Minor league pool            | **v4 only, optional.** Toggle at setup (off by default). ~10 generated players per team, can be promoted on injury or for late-season call-ups.                                                                                                                                                                                                                                                                          | Adds ~160 docs to a Standard preset; defer until budget measured.                                                                                                                                                                                |
| 21  | Award definitions            | **Composites.** MVP / Cy Young equivalents computed from raw stat inputs; the awards screen surfaces the math.                                                                                                                                                                                                                                                                                                           | v4 only.                                                                                                                                                                                                                                         |

## Persistence, replay, sharing

| #   | Topic                  | Decision                                                                                                                                                                                                                                                                                            | Notes                                                                                                                |
| --- | ---------------------- | --------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | -------------------------------------------------------------------------------------------------------------------- |
| 22  | Season archival        | **v4 only, only if measured doc count exceeds budget.** Recommendation when triggered: collapse a completed season's `seasonGames` + `transactions` + `seasonPlayerState` into a single compressed `seasonArchives` doc when the user starts a new season. Awards & final standings stay queryable. | Don't pre-build until we have measurements.                                                                          |
| 23  | Season replays         | **Test-only in v1, v2, and v3.** A user-facing "Replay this season" action ships in **v4 behind `featureFlags.allowReplay`**.                                                                                                                                                                       | Determinism must be proven across all v1–v3 systems (fatigue, injuries, trades, playoffs) before exposing replay UI. |
| 24  | Shareable season seeds | **Behind `featureFlags.shareableSeasonSeeds`.** Surfaces in the setup wizard's Advanced panel only when the flag is on.                                                                                                                                                                             | Default off. Flag is snapshotted to season doc on creation.                                                          |
| 25  | Mobile bracket UX      | **Vertical accordion** on mobile (each round expands → series → games). Horizontal tree on desktop.                                                                                                                                                                                                 | Goes with #17 playoff format.                                                                                        |

## Autogeneration

| #   | Topic                              | Decision                                                                                                                                                                                                          | Notes                                                                                                                  |
| --- | ---------------------------------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| A1  | Should we offer team autogen?      | **Yes, in v1.** Without it the user must hand-craft 8 / 16 / 24 customTeams before they can start a season — biggest UX unlock for v1.                                                                            |                                                                                                                        |
| A2  | Autogen default mode               | **Mixed.** User picks any subset; remaining slots autogenerated. Modes: Hand-pick · **Mixed** · All-autogenerated.                                                                                                |                                                                                                                        |
| A3  | Where do autogenerated teams live? | **In the user's `customTeams` library** — automatically promoted at setup time. They show up alongside hand-crafted teams, can be edited, exported, and reused.                                                   | Reaffirms decision #7. **No separate "league teams" collection.**                                                      |
| A4  | Naming themes                      | **All four ship in v1**: Classic (city + animal), Sci-fi (planet + role), Whimsical (food + verb), Random mix.                                                                                                    |                                                                                                                        |
| A5  | Autogen preview in wizard          | **Collapsed list with "Show all" expand.** Works at every preset size including Full (24 teams).                                                                                                                  | Recommendation accepted.                                                                                               |
| A6  | Per-team rename in wizard          | **v2.** v1 ships with rename-only at the team level (after season starts), via the existing customTeams editor. No per-player rename in v1.                                                                       | Reduces v1 wizard complexity.                                                                                          |
| A7  | Generator versioning               | Generator stores `autogen.version: 1` (nested under the `autogen` marker object) on each generated team's `customTeams` doc. Bump on any breaking generator change; surface a warning at replay time on mismatch. | Determinism contract for shareable seeds (v4). The nested marker shape is defined in [`data-model.md`](data-model.md). |
| A8  | Stat parity                        | Slider: **Balanced** (±5%) · **Mixed** (default, ±15%) · **Random** (no balancing).                                                                                                                               | Default "Mixed" so leagues have natural strong/weak teams.                                                             |

## Cross-cutting

| #   | Topic            | Decision                                                                                                                                                                                         | Notes                                             |
| --- | ---------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ | ------------------------------------------------- |
| X1  | RNG source       | All league-affecting randomness flows through `src/shared/utils/rng.ts` (mulberry32), seeded from the season master seed.                                                                        | No `Math.random()` anywhere in league code.       |
| X2  | ID generation    | Use `generateTeamId()`, `generatePlayerId()`, `generateSaveId()` from `@storage/generateId`. **No** `Date.now()` or `Math.random()` for IDs.                                                     | Repo-wide convention; reaffirmed for league code. |
| X3  | Hashing          | `fnv1a` from `@storage/hash`. Never reimplement.                                                                                                                                                 |                                                   |
| X4  | Schema evolution | Every schema bump ships a migration; same-version edits are forbidden. Indexed fields must appear in `required[]`. `migrationStrategies` lives on the collection-creator object, not the schema. | See `docs/rxdb-persistence.md`.                   |
| X5  | American English | All user-facing copy and docs use American English.                                                                                                                                              | Repo-wide convention.                             |

None of these block any v0–v4 PR; if any of them moves into v5+ planning, it gets its own design doc and decisions row.
