# League Mode — Setup Wizard

> Companion to [`README.md`](README.md). Defines the league-creation UX. v1 = Mini preset only; v2 + v4 add presets without changing the wizard's shape.

## Entry points

- **Home screen:** "Start a Season" CTA → `/leagues/new` (added in v1).
- **Leagues hub** (`/leagues`): "Create new season" button. Disabled with tooltip when an `active` season exists.

The wizard refuses to advance past Step 1 if `seasons` already contains an `active` document — it instead deep-links into the existing season's home (decision #8).

## UI reuse

Per [`ui-reuse.md`](ui-reuse.md), the wizard composes existing primitives — the modal shell extracted from the exhibition setup modal (`src/features/exhibition/styles.ts`, `max-height: min(90dvh, 820px)` desktop / `min(96dvh, 820px)` mobile — never bare `vh`), the field/label patterns already used in exhibition setup, and `StatusPill` (extracted in v1) for the autogen theme/parity selectors. Net-new wizard components are limited to `LeagueSetupWizard`, `AutogenPanel`, and the per-step renderers; everything underneath them is existing UI.

## Step layout

A single-page multi-step wizard (modal-on-route component). All steps are reachable backward; "Next" is disabled until the current step validates. State held in a local reducer; persisted to `sessionStorage` so a refresh does not lose progress.

```
Step 1 — Preset & length
Step 2 — Teams (selection + autogeneration)
Step 3 — League rules (DH per league, divisions [v2+])
Step 4 — Trade & playoff rules [v3+]
Step 5 — Seed & advanced
Step 6 — Review & create
```

In v1 only Steps 1, 2, 3, 5, 6 are visible. Step 4 appears starting in v3.

### Step 1 — Preset & length

- **League preset** (radio): Mini · Standard · Full. Disabled options show coming-soon badge ("Standard available in v2").
- **Season length** (radio): Sprint · Standard · Marathon. Disabled options follow the same convention.
- **Number of leagues**: 1 (default) or 2 in v1; v2+ supports up to 4 (Standard preset = 2 leagues × 2 divisions = 4 division groups).

Validation: at least one league.

### Step 2 — Teams

The most complex step. Three modes:

| Mode                  | Behavior                                                                                                       |
| --------------------- | -------------------------------------------------------------------------------------------------------------- |
| **Hand-pick**         | User selects N teams from `customTeams` (built-in + user-created). Wizard refuses to advance until N selected. |
| **Mixed** _(default)_ | User picks any subset; remaining slots autogenerated. Preview shown inline.                                    |
| **All autogen**       | One click → entire league of N teams autogenerated. Great for quickplay.                                       |

Layout (Mixed mode):

```
┌─ Step 2: Teams ──────────────────────────────────────┐
│ Need 8 teams. You've selected 3. Autogen 5.          │
│                                                       │
│ Mode: ⦿ Mixed   ○ Hand-pick   ○ All autogenerated    │
│                                                       │
│ My teams (3 selected): [Foxes✓] [Bears✓] [Sox✓] [+]  │
│                                                       │
│ ┌─ Autogenerate the rest (5 teams) ───────────────┐  │
│ │ Theme:  ⦿ Classic ○ Sci-fi ○ Whimsical ○ Mix    │  │
│ │ Parity: Balanced ─────●──── Random              │  │
│ │ Seed:   827314926     [🎲 Re-roll]              │  │
│ │ Preview: ▾ Show all 5 generated teams           │  │
│ └──────────────────────────────────────────────────┘  │
│                                                       │
│                                  [< Back]  [Next >]   │
└───────────────────────────────────────────────────────┘
```

Validation:

- Selected + autogen count must equal preset team count.
- Each selected team must meet **league-play roster minimums** (decision #13: v1 = lineup 9 + bench 3 + 5 SP + 3 RP = 20 active; v2+ bumps bench to 5 and RPs to 4 = 23 active). The wizard warns when a hand-picked team is below the target and offers the auto-fill action below.
- Hand-picked teams below the league-play minimum show an "Auto-fill missing slots" affordance that appends autogenerated bench/pitchers **into the season snapshot only**, never into the user's persistent `customTeams` doc.

Autogenerated team output is held in **wizard reducer state only** until Step 6 fires the single transactional create. No `customTeams` writes happen at Step 2, so abandoning the wizard mid-flow leaves no orphan docs and no duplicate-key risk on retry. (Re-roll, theme/parity changes, and back-navigation are pure in-memory operations against the autogen seed.) The persistent write is described in Step 6 below.

### Step 3 — League rules

- **Per-league DH** (radio per league): On / Off.
  - Quick-presets at the top: "Classic AL/NL" · "Universal DH" · "Pitchers Hit" · "Custom".
  - For 1-league seasons, the preset row collapses to a single On/Off toggle.
- **Divisions** (v2+): per league, "No divisions" / 2 / 3 / 4. Distribution preview shown.
- **Series length** (radio): 3 (default) · 2 · 4.

### Step 4 — Trade & playoff rules (v3+)

- **Trade deadline**: slider 50% – 100% of season + "No deadline" checkbox. Default 70%.
- **AI trade aggressiveness**: Passive · **Moderate** (default) · Active.
- **Playoff format**: Default (WC bo3 / LCS bo5 / BCS bo7) · Short (1/3/5) · Long (5/7/7).

### Step 5 — Seed & advanced

- **Master seed**: text field, pre-populated with `generateFreshSeed()`. "Re-roll" button.
- **Advanced** (collapsed by default):
  - "Share this season seed" toggle — visible only when `featureFlags.shareableSeasonSeeds` is enabled.
  - "Enable minor leagues" toggle — visible only in v4+.
  - "Multi-team manager mode" toggle — visible only in v4+.

### Step 6 — Review & create

Summary panel showing all selected options grouped by step. Big "Create Season" button at the bottom. Clicking it runs **a single transactional create** (this is the only persistence point in the wizard — Steps 1–5 are all in-memory):

1. Generates ids for season, season-teams, season-games, etc.
2. Writes any autogenerated `customTeams` docs that the wizard built in Step 2 (idempotent insert; no-op if user used Hand-pick mode). Per decision #7 / #A3 these are real `customTeams` docs from this point on, visible in the user's library.
3. Writes the `seasons` doc.
4. Generates the schedule (see [`schedule-and-sim.md`](schedule-and-sim.md)).
5. Writes `seasonTeams`, `seasonGames`, initial `seasonPlayerState`.
6. Navigates to `/leagues/:seasonId`.

All writes are wrapped in a single logical transaction; failure halfway through deletes any partially-created docs (including the autogenerated `customTeams` rows from step 2). Once the season is committed, the **roster-edit lock engages** for those `customTeams` docs (see [`data-model.md`](data-model.md)).

## State machine

Setup wizard state is reducer-driven, mirroring `editorReducer` in `CustomTeamEditor`. Actions:

```
SET_PRESET, SET_SEASON_LENGTH, SET_LEAGUE_COUNT,
SET_TEAM_MODE, TOGGLE_TEAM_SELECT, SET_AUTOGEN_THEME,
SET_AUTOGEN_PARITY, REROLL_AUTOGEN_SEED, AUTOFILL_TEAM_SLOTS,
SET_DH(leagueIndex, on|off), APPLY_DH_PRESET,
SET_DIVISION_COUNT, SET_SERIES_LENGTH,
SET_TRADE_DEADLINE, SET_TRADE_AGGRESSIVENESS, SET_PLAYOFF_FORMAT,
SET_MASTER_SEED, REROLL_MASTER_SEED, TOGGLE_FLAG(flagKey)
```

A pure `validateWizardState(state) → string[]` returns the list of validation errors blocking the current step's "Next" button. Validation is exhaustively unit-tested.

## Accessibility

- All step navigation is keyboard-accessible.
- Each form field has a visible `<label>` (per UI style guide).
- Validation errors render in a live region (`aria-live="polite"`).
- "Re-roll" buttons announce the new seed value.

## Testing surface

- Unit: reducer + validator coverage (every step's required fields, every error path).
- Component: render every step in every mode; assert correct controls visible.
- E2E: `e2e/tests/league-setup.spec.ts` (added in v1) walks the wizard end-to-end, picks Hand-pick + Mixed + All-autogen modes, and asserts navigation to `/leagues/:seasonId`. A second E2E asserts that creating a season twice with the same master seed produces identical `seasonGames.derivedSeed[]` arrays.
