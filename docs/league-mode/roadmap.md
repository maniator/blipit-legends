# League Mode — Roadmap (v0 → v4)

> Companion to [`README.md`](README.md). This doc enumerates **what each version ships**, **what it explicitly defers**, and the dependency order between phases. Locked design decisions live in [`decisions.md`](decisions.md).

**Goal:** establish a single, coherent design for League Mode before any implementation. Replace every previous `docs/league-mode/*.md` draft with the docs listed in [`README.md`](README.md).

**Ships:**

- The full doc set under `docs/league-mode/` (this version).
- Update to `docs/README.md` index pointing at the new league-mode README.
- No source code changes. No schema changes. No new dependencies.

**Acceptance:**

- All previous `docs/league-mode/*.md` files are removed.
- New docs cover scope, decisions, data model, setup, autogeneration, scheduling, fatigue, playoffs/trades, awards/archive, routing, and risks.
- `docs/README.md` league-mode link still resolves.
- `yarn format:check` passes on all `.md` files.

**Defers:** everything else.

## v1 — Foundation + pitcher fatigue ("Watch a season")

**Goal:** make a complete, end-to-end "create a small league → watch a 30-game season → see standings → declare a champion" loop, with realistic pitcher rotation behavior.

**Ships:**

- **Setup wizard** (see [`setup-wizard.md`](setup-wizard.md)) supporting:
  - Mini preset only (8 teams).
  - Single league or two small leagues (4 teams each).
  - 30-game season.
  - Per-league DH toggle (defaults: AL on / NL off, Universal DH, Pitchers Hit).
  - Team selection: hand-pick, mixed, or all-autogenerated. **All four naming themes** ship in v1 (Classic, Sci-fi, Whimsical, Random mix).
  - Master seed (random by default).
- **Team autogeneration** (see [`team-autogeneration.md`](team-autogeneration.md)) — generated teams are **promoted into the user's `customTeams` library** automatically. There is no separate "league teams" collection.
- **New RxDB collections** (v1 of each schema; see [`data-model.md`](data-model.md)):
  - `seasons`
  - `seasonTeams`
  - `seasonGames`
  - `seasonPlayerState` — pitcher availability fields only (no `wear`, no `injury`).
- **Schedule generation** — deterministic round-robin, series-based, byes for odd team counts.
- **League hub UI** — list of seasons (active + completed), season home (standings, schedule, "next game" CTA).
- **Team page** in season context — read-only lineup view with pitcher fatigue badge.
- **Game launch** → existing `GamePage` with league context and minimal `seasonModifiers.pitcher` injection.
- **Headless "sim to next user game"** loop — sequential, RNG-deterministic, idempotent.
- **Pitcher fatigue (v1 form)**:
  - Rest-day tracking on `seasonPlayerState`.
  - Up to ~12% effect knob (capped, exact magnitude trusted to design defaults per [`decisions.md`](decisions.md) #11).
  - Pill UI badge: `🟢 Fresh` / `🟡 Tired` / `🔴 Spent` on player rows in lineup card, season team page, and manager-mode prompts.
  - Ineligibility-to-start enforcement (a "Spent" SP cannot be the starter).
- **Result write-back + season tick** — pitcher availability only.
- **Persistence** — full save/resume of in-flight season; deterministic replay verified by tests.
- **Manager Mode in v1**: user team only. Other teams use existing AI strategy.

**Defers:**

- Standard / Full league sizes.
- 60-game / 120-game seasons.
- Position-player wear and injuries.
- Trades, playoffs, awards.
- Multi-team manager mode.
- Minor leagues, replay UI, shareable seeds (test-only determinism is enforced).

**Acceptance:**

- A user can complete a full Mini-preset season end-to-end with both watch and quick-sim modes.
- Pitcher fatigue measurably affects who is eligible to start.
- Same master seed reproduces an identical season log (verified by an E2E test).
- Standings update on every completed game; champion declared at game 30.

## v2 — Realism: full rosters, wear, injuries

**Goal:** Standard preset (16 teams, 2 leagues × 2 divisions, 60 games), full fatigue/injury system, in-depth roster management.

**Ships:**

- **Standard preset** in setup wizard.
- **2 leagues × 2 divisions** layout.
- **Interleague schedule** with DH harmonization (home team's rule).
- **Position-player wear** added to `seasonPlayerState` (~5% knob) — schema bump on `seasonPlayerState`.
- **Injury system**: 1.5% per active-lineup player-game (only active lineup rolls, bench does not), IL collection, automatic bench-fill replacement, return rolls per recovery distribution. The rate is held in a `regularSeasonInjuryRatePctPerGame` constant pinned to `seasons.rulesetVersion` so post-launch tuning does not break replay determinism for in-flight seasons. See [`fatigue-and-injuries.md`](fatigue-and-injuries.md) for the canonical value.
- **`seasonTransactions` collection introduced in v2** to back the IL events and the new transactions feed UI. Schema is forward-compatible with v3 trade kinds — no v2→v3 schema bump on this collection. (Promoted from v3 per the PM-agent dev-readiness review.)
- **Injury report UI**, **transactions feed UI**.
- **Expanded league-play roster minimums**: bench ≥ 5, pitchers ≥ 9 (lineup 9 + bench 5 + 5 SP + 4 RP minimum = 23). Setup wizard offers to "auto-fill missing slots from autogen" for hand-picked teams below this minimum, **without** mutating the user's persistent `customTeams` doc — only the season snapshot is augmented for that case.
- **In-depth roster management UI** — set lineup, set rotation, manage IL, swap bench.
- **AI rest behavior** — pitchers gated by strict eligibility threshold (SP ≥ 0.70, RP ≥ 0.35); position players use the **soft probability** formula `restProb = clamp((wear − 6)/4, 0, 1)` rolled after lineup but before box-score sim, broken ties by lex `playerId`. See [`fatigue-and-injuries.md`](fatigue-and-injuries.md) and decisions #12/#13.

**Defers:** trades, playoffs, awards, minors, multi-team manager mode, replay UI.

## v3 — Postseason & trades

**Goal:** real season meaning — pennant races, deadline drama, October baseball.

**Ships:**

- **Trade engine** (see [`playoffs-and-trades.md`](playoffs-and-trades.md)):
  - Manual trades from team page (multi-asset, salary-free).
  - AI ↔ AI trades. Default ~2–3 per team per season ("moderate"); selectable passive / moderate / active.
  - **No user veto** on AI ↔ AI trades.
  - Trade deadline: default 70% of season, configurable 50%–100% or "no deadline" at setup time. Past the deadline, all trade UIs are read-only.
  - Transactions feed surfaces every trade.
- **Playoff bracket**:
  - 4 teams per league.
  - Default series lengths: WC bo3 / LCS bo5 / BCS bo7. Presets: "Short" (1/3/5), "Long" (5/7/7).
  - Reseed after each round by regular-season record.
  - Tiebreakers per decision #18 (head-to-head → run differential → seeded coin flip).
- **Playoff fatigue/injury rules**: full pitcher fatigue (more important here, not less); injury rate scaled to **×0.5** of regular season (0.75% per active-lineup player-game). Position-player wear unchanged.
- **Bracket UI**: vertical accordion on mobile, horizontal tree on desktop.
- **Tiebreaker chain (decision #18, full operationalization):** head-to-head record → intra-division record (v2+) → run differential vs the tied group → full-season run differential → seeded-PRNG coin flip from the `:tiebreak:${tieKey}` sub-stream. Three-way+ ties resolve highest-seed-first then recurse. Deterministic, never `Math.random()`.

**Defers:** awards screen, minor leagues, multi-team manager mode, replay/seed-sharing UI.

## v4 — Polish, awards, full preset

**Goal:** "complete game" experience — long seasons, awards ceremony, optional carryover.

**Ships:**

- **Full preset** (24 teams, 120-game marathon) added to setup wizard.
- **Awards screen** — composite formulas (MVP, Cy Young equivalents) computed from `seasonPlayerState` + `seasonGames`. Each award shows raw inputs ("OPS-proxy: .823 · Team W-L bonus: +12") so the user sees the math.
- **Leaders screen** — season leaders by stat.
- **Optional minor-league call-ups** — additional ~10 generated players per team, can be promoted on injury or for late-season rosters. Toggleable at setup ("Enable minor leagues" — off by default).
- **Optional minimal offseason carryover** — toggle at "Start New Season": heal IL, reset fatigue, keep team rosters and league membership.
- **"Sim full season" mode** with a progress bar that respects a per-tick fairness budget (no UI freeze).
- **Multi-team manager mode** — toggle in setup; lets the user manage every team. Off by default.
- **Season archival** — _only if measured doc count exceeds budget_ (see [`awards-and-archive.md`](awards-and-archive.md)). When triggered, completed `seasonGames` + `transactions` + `seasonPlayerState` for that season collapse into a single compressed `seasonArchives` doc; awards and final standings stay queryable.
- **Feature-flagged**:
  - `featureFlags.shareableSeasonSeeds` — exposes "Share this season seed" UI in the wizard's Advanced panel.
  - `featureFlags.allowReplay` — exposes "Replay this season" action on completed seasons.
  - Both default off.

**Defers:** anything not on this list.

## Phase dependencies

```
v0 (docs)  ──►  v1 (foundation + pitcher fatigue)
                 │
                 ├──►  v2 (rosters + wear + injuries)
                 │       │
                 │       └──►  v3 (playoffs + trades)
                 │               │
                 │               └──►  v4 (polish + awards + archive)
                 │
                 └──►  (some v4 surfaces only need v1, e.g. seed-sharing flag wiring,
                         but full v4 sits on top of v3)
```

- v2 _must_ ship before v3 (playoff & trade UI both depend on roster management surfaces and the injury feed).
- v4 awards depend on stats accumulated by v2 (`seasonPlayerState.wear`, `seasonGames` extended box scores).

## Cross-phase invariants

These are reaffirmed in [`README.md`](README.md) but worth repeating per phase:

- All teams (built-in, custom, autogenerated) live in **one `customTeams` collection**. League state references them by id.
- Per-game RNG seed derivation never changes shape across versions. v1 and v4 must produce the same seed for the same `(seasonId, seasonGameId)`.
- Every schema change bumps `version` and ships a migration. No same-version schema rewrites.
- Feature-flag snapshot on the season doc — flipping a flag mid-season is a no-op until the next season starts.
