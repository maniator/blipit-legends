## Locked decisions to enforce in v2

| Topic                           | Value                                                                                                                  |
| ------------------------------- | ---------------------------------------------------------------------------------------------------------------------- |
| Preset                          | Standard (16) added; Mini still supported                                                                              |
| Season length                   | Standard (60) added; Sprint still supported                                                                            |
| Schedule                        | Round-robin + interleague; DH harmonization = home team's rule                                                         |
| Position-player wear            | `wear: 0..10` field; ~5% effect knob                                                                                   |
| Wear recovery                   | Bench player wear ticks down 0.5/rest day, floored at 0; off-day decays everyone                                       |
| Injury rate                     | 1.5% per active-lineup player-game (regular season). Pinned to `rulesetVersion`                                        |
| Injury kind/duration            | Short 60% (1–3) / Medium 30% (4–10) / Long 10% (11–30); kind THEN duration draw order                                  |
| Injury PRNG                     | Per-game `derivedSeed` PRNG, **after** box-score sim, in fixed iteration order                                         |
| Recovery clock                  | At **start** of game-day (binding for replay)                                                                          |
| AI rest (position players)      | **Soft probability** `restProb = clamp((wear − 6)/4, 0, 1)`; rolled after lineup, before box-score sim, lex `playerId` |
| AI rest (pitchers)              | Strict eligibility threshold (unchanged from v1)                                                                       |
| RP cumulative load              | (carryover from v1) hard cap 3 consecutive days; 4-day load multiplier                                                 |
| League-play roster minimum (v2) | Lineup 9 + bench 5 + 8 pitchers (5 SP + 3 RP) = 22 active                                                              |
| `seasonTransactions` kinds      | `il_in`, `il_out`, `inconsistency_detected` ship in v2; `kind` declared as `string` (not enum) for forward compat      |
| Catcher wear                    | Universal `wear += 1/game` ships as known v2 limitation; retune tracked in `follow-ups.md` F-6                         |
| `rulesetVersion`                | Bump to `2` ONLY if any v1 constant changed; otherwise stays `1` and v2 ships an additive ruleset                      |

## Non-negotiable contracts

- All v1 contracts hold (sub-PRNG isolation, watch=headless PRNG parity, single-bulk-write per game, capability-Symbol for sanctioned writes, etc.).
- **`rulesetVersion` gates phases, not just constants.** v1 seasons opened on the v2 build do **not** run injury rolls. The simulation entry point dispatches on `season.rulesetVersion` to a frozen phase pipeline; adding a phase requires bumping `rulesetVersion` and adding a new pipeline entry.
- **Schema bumps with migration strategies:**
  - `seasonPlayerState` v1→v2 — add `wear`, `injuryStatus`, `seasonStats` with documented defaults.
  - `seasons` v1→v2 — add `divisions[]` and (if needed) `rulesetVersion`-related fields. Backfill `divisions` as `[{ id:'div_default', name:'', teamIds: leagues[].teamIds }]`.
  - `customTeams` (if v2 adds anything) — never throws, additive, optional fields use the "key omitted" pattern (not `undefined`).
- **`seasonTransactions` write IDs are deterministic:** `il:${seasonPlayerStateId}:${gameDay}:il_in` and `il:${seasonPlayerStateId}:${gameDay}:il_out`. Re-running a day is a no-op.
- **`seasonModifiers.injuryFilter`** is reconstructed from `seasonPlayerState` on every game launch; **never persisted**. Document this in the type's docstring so future work doesn't try to serialize it.
- **`seasonModifiers` shape is frozen.** v2 may only add fields, never rename or remove. Enforced by the type-snapshot CI test tracked in `follow-ups.md` F-14.
- **In-depth roster management is sanctioned-write only** — the lineup / rotation editor in season UI uses `withSanctionedCustomTeamWrite('lineup', …)`; it writes to `seasonTeams.rosterSnapshot`, never to `customTeams`.

## Acceptance criteria

- A full Standard 16-team / 60-game / 2-leagues × 2-divisions season completes end-to-end via watch + quick-sim.
- Average injuries per team per Standard season fall within `9 ± 3` at default constants.
- IL fills automatically from bench; transactions feed shows every IL move; injury report screen lists all currently-IL players.
- Replay across two runs of the same `masterSeed` is byte-identical for `seasonGames.derivedSeed[]`, the play-by-play, **and** `seasonTransactions[]` (chronological order included).
- Loading a v1 save in v2 keeps the season on `rulesetVersion: 1` (no injury rolls retroactively applied).
- "Auto-fill missing slots from autogen" augments only the season snapshot; the user's persistent `customTeams` doc is unchanged.
- Setup wizard surfaces division configuration; lineup builder reads DH from the home team's league.
- All new schema bumps ship an upgrade-path unit test.
- All new screens have mobile + desktop visual baselines via `@e2e-test-runner` in the Playwright Docker container.

## Risks to watch (from `risks.md`)

## Testing surface (must ship in v2 PR)

- **Unit:** injury distribution sampler (kind/duration); IL clear-on-recovery; wear floor/ceiling; soft-rest probability formula; RP cumulative load enforcement; ruleset-version dispatch.
- **Schema:** upgrade-path test for `seasonPlayerState` v1→v2 and `seasons` v1→v2; loading a v1 save renders without injury rolls.
- **Integration:** Standard 60-game season produces injury distribution within target band over N seeds.
- **E2E:** same `masterSeed` produces identical `seasonGames.derivedSeed[]` AND identical `seasonTransactions[]`; injury report + transactions feed visual snapshots.

## Out of scope for v2

Trades, playoffs, awards, minors, multi-team manager mode, replay UI, shareable seeds.
