## Non-negotiable contracts

- **Single team library.** All teams (built-in, custom, autogenerated) live in the existing `customTeams` collection. League state references them by id.
- **`customTeams` schema is currently at version 0** (reset to v0 in the v1.2 beta epoch; prior migration history is preserved in git history only). Adding `activeLeagueIds` and `autogen` fields requires bumping it to **version 1** with an identity migration (`migrationStrategies[1]: (d) => d`). Do not rely on `docs/rxdb-persistence.md`'s historical note about "version 3" — that predates the beta reset.
- **Roster-edit lock.** Storage-layer write guard rejects mutations to `customTeams` docs in any `status='active'` season unless tagged with the sanctioned-write capability Symbol (see `../data-model.md`).
- **Seed contract.** Per-game seed = `fnv1a(\`${seasonId}:${seasonGameId}\`) → uint32 → base-36`, cached on `seasonGames.derivedSeed` at schedule generation, **immutable across migrations**.
- **Sub-PRNG isolation.** Schedule = `mulberry32(parseInt(fnv1a(\`${masterSeed}:schedule\`), 16) >>> 0)`. Per-game = cached `derivedSeed`. Never share streams. The `parseInt(..., 16) >>> 0`step converts`fnv1a`'s hex output to the `uint32`that`mulberry32`requires — passing the raw hex string silently breaks determinism. (See full registry in`../schedule-and-sim.md`.)
- **`reinitSeed` is the LAST step** before sim entry; ESLint-restricted import; only the simulation entry point may call it.
- **Watch ↔ headless mutex** + per-`seasonId` batch mutex + per-game writer-id claim; multi-tab stale-claim recovery.
- **Per-game flow is a single bulk write** (`seasonTransactions` first → `seasonPlayerState` → `seasonGames`); idempotent on `seasonGameId`; aggregates derived (recomputed by querying `seasonGames`), not incremented.
- **Watch = headless PRNG parity.** AI rest + injury rolls run for both teams in every game regardless of mode; manager-mode UI only surfaces decisions, never substitutes draws.
- **`/game?seasonGameId=…`** query-string handoff (NOT path, NOT `location.state`) for league watch-mode entry.
- **All randomness via `src/shared/utils/rng.ts`.** Zero `Math.random()` in league code (ESLint guard).
- **All IDs via `@storage/generateId`.** Add `generateSeasonId()`, `generateSeasonGameId()`.
- **Schema versioning.** Every collection bumps `version` and ships a `migrationStrategies` entry (never throws, sync, pure). Indexed fields appear in `required[]`. See ripple table in `../roadmap.md`.

## Acceptance criteria

- A user can complete a full Mini-preset Sprint season (single league, 1 league × 8 teams × 30 games, or two leagues × 4 teams × 30 games) end-to-end via watch and quick-sim modes.
- Pitcher fatigue measurably affects who is eligible to start; SP starts respect the eligibility threshold; the per-appearance availability curve naturally rate-limits RP usage.
- Same `masterSeed` reproduces an identical season log (verified by an E2E test asserting bit-identical `seasonGames.derivedSeed[]`, identical play-by-play, identical final standings).
- Standings update on every completed game; champion declared at game 30.
- Custom Team Editor renders in `mode="locked"` for any team in the active season; storage-layer guard rejects unsanctioned writes; route-level UX shows the locked banner with deep-link to season's team page.
- Reload-safe: `/game?seasonGameId=…` rehydrates from RxDB; cold deep-link to `/leagues/:seasonId/...` works.
- Save bundle exports include league state (per `../save-export.md`); import round-trips a mid-season league with no data loss.
- All new colors, font sizes, and component variants live in `docs/style-guide.md` (or its in-PR addition); no inline hex strings.
- New schemas hit indexes for every documented query path (`seasons.find({selector:{status:'active'}})`, `seasonGames` by `(seasonId, gameDay)`, `seasonGames` by `(seasonId, status)`, `seasonPlayerState` by `(seasonId, seasonTeamId)`).
- **Style-guide additions land in `docs/style-guide.md`** (canonical edit owned by `@ux-design-lead`): the 5 League Mode tokens (`StatusPill`, `StatusBanner`, `EmptyState`, `BracketConnector`, `SortableTable`) per [`style-guide-additions.md`](../style-guide-additions.md).
- **Pill color contrast measurements** against `bgSurface` for the three pill colors (`#ff6b6b`, `#ffd06b`, `#a0b4d0`) — owner `@ux-design-lead`. Numbers must land before the v1 visual-snapshot baseline.
- **`@user-save-curator` proxy interview** on the `/saves` vs `/leagues` mental-model split — owner `@ux-design-lead`. Validates that the nav rename ("Exhibitions" vs "Leagues") + home-screen "Continue" CTA resolves the friction surfaced in round-2 UX review.
- **ESLint rule restricting `reinitSeed` imports** to exactly two permitted module paths: `src/features/gameplay/components/Game/GameInner.tsx` (existing) and `src/features/league/sim/leagueSimRunner.ts` (the planned league entry point). Any other importer of `reinitSeed` fails the lint check — owner `@simulation-correctness`. The seed contract in [`schedule-and-sim.md`](../schedule-and-sim.md) names the rule; v1 PR either lands the rule or opens a tracked follow-up issue.
- **Quick Start path** (per `@user-casual-watcher` proxy interview): the leagues hub surfaces a single "🎲 Quick Start" CTA on the empty state that bypasses the 6-step wizard and creates a Mini-Sprint season with sensible defaults (8 autogenerated teams, mixed parity, mixed naming themes, fresh seed). The wizard remains the path for users who want control. Owner `@ux-design-lead`.
- **Self-labeling Continue chip** (per `@user-save-curator` and `@user-casual-watcher` interviews): the home-screen "Continue" CTA shows what's being resumed, not just "Continue" — e.g., "▶ Continue: 2026 Backyard League · day 12/30" or "▶ Continue: Yankees vs Red Sox · top of 6". Owner `@ux-design-lead`.
- **Locked-team deep-link target is functional, not read-only** (per `@user-custom-team-builder` and `@user-manager-strategist` interviews): the deep-link from the locked-mode banner lands on a season-context team page where the user can reorder lineup, set rotation order for the next series, and designate bullpen roles against the `seasonTeams.rosterSnapshot` (sanctioned-write capability). v1 must ship this page — without it the lock feels like the team is hostage. Identity-of-roster (which players) stays locked; **order and roles** are editable.
- **"Auto" badge in Custom Teams list** (per `@user-custom-team-builder` interview): teams with `autogen.version` set render a small `<StatusPill variant="auto">` next to the team name in `/teams`, visible by default (not behind a filter). Surfaces the theme too where it fits ("Auto · Sci-fi"). Bulk-select + delete is also v1 scope to keep library bloat manageable; the lock continues to prevent deleting a team in any active season.
- **Save / import error microcopy** (per `@user-save-curator` interview): every import-rejection path (FNV-1a checksum mismatch, ruleset-version too new, active-season collision) ends its message with the explicit reassurance "Your existing saves were not changed." Post-import shows a single summary modal ("3 teams overwritten, 2 added, 1 season imported as 'X (imported)'"), not per-doc prompts. Owner `@ux-design-lead` + `@rxdb-save-integrity`.

## Testing surface (must ship in v1 PR)

- **Unit:** schedule determinism across team-count × series-length × seed; no-double-booking invariant; `deriveScheduledGameSeed` round-trip; idempotent completion; pitcher recovery curve; sanctioned-write capability rejection.
- **Schema:** upgrade-path test for every collection bump (open at old version, insert legacy doc, reopen, assert all fields survive).
- **Integration:** sim-to-next-user-game advances `currentGameDay` correctly; SaveStore export/import preserves a mid-season league.
- **Integration:** import-recovery contract — simulate an interrupted import (kill the process between `customTeams` step 3 and `seasons` step 4 of the import order in `save-export.md`) → reboot → assert the `importInProgress` tombstone scan surfaces a "previous import was interrupted" notification AND the wipe branch removes the orphaned `customTeams` rows cleanly. No silent partial-state.
- **E2E:** same `masterSeed` produces identical `seasonGames.derivedSeed[]` and standings; locked-team editor visual snapshot (mobile + desktop); leagues hub empty/active/complete state visual snapshots (mobile + desktop).

Position-player wear, injuries, IL, trades, playoffs, awards, minors, multi-team manager, replay UI, shareable seeds (test-only determinism only). See `../roadmap.md` v1 "Defers" + the global "won't ship in v1–v4" register in `../decisions.md`.
