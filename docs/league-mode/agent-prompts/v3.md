## Locked decisions to enforce in v3

| Topic                        | Value                                                                                                                                                                     |
| ---------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Trade deadline               | Default = 70% of season; configurable 50%–100% or "no deadline" at setup                                                                                                  |
| AI trade aggressiveness      | Passive 0–1, **Moderate** (default) 1–2, Active 3–4 trades/team/season; clusters near deadline (50%+ of trades in last 20% pre-deadline)                                  |
| User veto on AI ↔ AI trades  | None — surfaced in transactions feed only                                                                                                                                 |
| Trade scheduler cadence      | Once per `gameDay`, **after** start-of-day IL recovery and **before** any games sim                                                                                       |
| Trade scheduler iteration    | Lex-sorted `seasonTeamId`; one accepted trade per team per cadence; once a player moves they cannot move again same day                                                   |
| Trade scheduler sub-PRNG     | `mulberry32(parseInt(fnv1a(\`${masterSeed}:trades:day:${gameDay}\`), 16) >>> 0)`                                                                                          |
| Tiebreaker chain             | H2H pairwise → intra-division (v2+) → run-diff vs tied group → run-diff full-season → coin flip from `:tiebreak:${tieKey}` sub-PRNG; iterative for 3-way+                 |
| Playoff format               | 4 teams/league; WC bo3 / LCS bo5 / BCS bo7 (default); presets Short (1/3/5), Long (5/7/7)                                                                                 |
| Reseeding                    | After each round, by regular-season record + tiebreaker chain                                                                                                             |
| Home-field advantage         | Higher seed hosts game 1, game 2, and any deciding game N (MLB 2-2-1 / 2-3-2 patterns); explicit per-series-length host pattern                                           |
| Playoff fatigue              | **Full strength**, same recovery + eligibility                                                                                                                            |
| Playoff injury rate          | ×0.5 of regular season (decision #19)                                                                                                                                     |
| Playoff position-player wear | Unchanged                                                                                                                                                                 |
| Bracket persistence          | Bracket structure (seeds + tiebreaker outcomes) persisted to `seasons.bracket` (or `seasonBrackets` collection) at end of regular season                                  |
| Bracket UI                   | Vertical accordion (mobile, native `<details>`) / horizontal tree (tablet+desktop), via `mq.mobile`/`mq.notMobile`                                                        |
| `rulesetVersion`             | Bump to `3` only if any v1/v2 constant changes during v3 implementation; the ×0.5 playoff multiplier is the v1 value (decision #19) and does not by itself require a bump |

## Non-negotiable contracts

- All v1+v2 contracts hold.
- **Trade engine determinism.** Sub-PRNG isolation (`:trades:day:`); lex-sorted iteration; one-accepted-trade-per-team-per-day cap; once-moved-cannot-move-again-same-day rule (no A→B→C→A churn); deterministic `proposalId = fnv1a(seasonId + gameDay + sortedTeamPair + sortedPlayerIds)` makes re-running a day a no-op.
- **Trade execution atomicity.** Per `../playoffs-and-trades.md`: update each `seasonPlayerState.seasonTeamId`, update both `seasonTeams.rosterSnapshot`, append `seasonTransactions` row, all under the per-game bulk-write contract. Roll back on partial failure via RxDB document-version checks.
- **Tiebreaker PRNG isolation.** Coin-flip uses `mulberry32(parseInt(fnv1a(\`${masterSeed}:tiebreak:${sortedTiedIds}\`), 16) >>> 0)`. Same `tieKey` always produces the same flip whether evaluated for standings, seeding, or reseed.
- **Three-way+ ties** resolved iteratively: highest seed first → remove → repeat with the residual group's `tieKey`.
- **Bracket persistence** prevents re-derivation drift if any tiebreaker, ruleset, or sub-PRNG convention shifts later. Resume reads `seasons.bracket`; never re-runs the seeding pass.
- **`TradeProposalDialog` ≠ `DecisionPanel`.** Trade UI has no time pressure; uses a side-by-side roster-comparison dialog. Borrow modal shell only.
- **Watch ↔ trade interaction.** Trade offers arriving while a watch-mode game is in-flight are **queued**, surfaced after the game completes. (Documented in `../playoffs-and-trades.md`.)
- **`seasonTransactions.kind`** stays `string` (not RxDB enum). Trade kinds (`'trade'`, `'trade_aborted'`) are validated at the app layer.

## Acceptance criteria

- A 16-team Standard 60-game season completes regular season + playoffs end-to-end.
- Average AI ↔ AI trades per team per Standard season fall within configured aggressiveness band ± 1; trade clustering near deadline is observable.
- Tiebreaker test suite covers: 2-way H2H tie; 2-way H2H + run-diff tie; 3-way H2H tie (rock-paper-scissors); 4-way tie at the wildcard cliff. All deterministic across two runs of the same seed.
- Bracket reseeds correctly after each round across all tie cases.
- Trade rollback test on simulated RxDB write failure leaves all docs consistent.
- Replay across two runs is identical including trade timing, bracket seeding, and series outcomes.
- Bracket UI: mobile accordion + desktop tree, both visual-snapshotted; keyboard navigation works on the desktop tree (Tab → next series; arrows within series; `aria-current="round"` on active round).
- Loading a v1 or v2 save in v3 keeps the season on its original `rulesetVersion` — no playoffs are forced onto a v1/v2 season.
- **Post-v3 trade-outcome audit** (`@baseball-manager`) on the first real Standard season log — verifies symmetric scoring prevents egregious "stars shipped for nothing" trades.
- **Live trade volume / temporal distribution validation** (`@baseball-manager`) on the same season log — confirms clustering near deadline (50%+ of trades in last 20% pre-deadline).

## Testing surface (must ship in v3 PR)

- **Unit:** trade evaluation deterministic across aggressiveness; deadline enforcement; trade rollback on partial-write; tiebreaker correctness across H2H / run-diff / coin-flip; reseeding pairings.
- **Integration:** Standard season with "moderate" AI runs trades within band across 100 simulated runs; deterministic 16-team Standard season → bracket generation produces identical results across runs with the same seed.
- **E2E:** bracket UI on mobile + desktop; series advancement; trade rejection when deadline passes; visual snapshots of bracket at "all rounds complete".
