## Non-negotiable contracts

- All v1+v2+v3 contracts hold.
- **Awards are pure functions** of `seasonPlayerState.seasonStats` + `seasonTeams` totals + `seasons` snapshot fields. Same season state → same awards.
- **Award formulas are pinned to `rulesetVersion`.** A v4.1 retune cannot retroactively change a season's stored `seasonAwards.formula` breakdown.
- **`seasonArchives` integrity:** `checksum: string` (FNV-1a of raw JSON pre-gzip) verified on read. Mismatch logs `data-integrity` error and surfaces "archive corrupted — historical detail unavailable" without deleting (manual recovery path preserved). `seasons` and `seasonAwards` stay queryable.
- **Archive write order:** insert `seasonArchives` doc → re-read to confirm → only then delete source docs. RxDB has no multi-collection transactions; this order is the recovery contract.
- **Carryover order of operations:** (1) mark previous season `complete`/`abandoned` (releases lock); (2) apply carryover write-back to `customTeams` via sanctioned-write capability; (3) begin new season setup. Step 1 always precedes step 2.
- **Multi-team manager** lifts `teamIds` from length 1 to length N **without rewriting** the v1 manager-mode prompt surface (the v1 refactor anticipated this).
- **Cancellation of "Sim full season"** keeps every completed game; the season stays in a consistent state. Idempotency contract makes this safe.
- **`activeLeagueIds` cache** stays accurate via subscription to `seasons.$` (per `data-model.md` lock-cache invalidation rule); v4 carryover is the only sanctioned writer that mutates `customTeams` post-season.

## Acceptance criteria

- A Full 24-team / 120-game Marathon season completes inside the per-tick budget without UI freeze.
- Awards screen renders the full formula breakdown for every winner ("OPS-proxy: .823 · Team W-L bonus: +12 → composite 0.711, rank 1 of 156").
- Award PA / IP floors reject under-qualified candidates; RoY shows "No eligible rookies this season" empty state when applicable.
- Carryover preserves rosters + clears IL/fatigue + resets `pitcherDaysRest`/`pitcherAvailability`/`wear`; league membership preserved.
- Carryover **never** writes to a locked `customTeams` doc (asserted by unit test).
- Archive round-trip is lossless **and** detects corruption via FNV-1a checksum mismatch (asserted by injecting a single-byte mutation into `compressed`).
- Multi-team manager mode: when N>1 teams have simultaneous decisions on the same game-day, prompts route per `multi-team-manager.md` rule (queueing or primary-team priority).
- "Sim full season" can be cancelled mid-batch; resuming completes only the unfinished games.
- Feature-flag snapshot: flipping a flag mid-season is a no-op until the next season starts.
- Loading a v1/v2/v3 save in v4 keeps the season on its original `rulesetVersion` — no v4 phases (awards, minors) are forced onto a pre-v4 season.

- **Unit:** every composite formula handles edge cases (zero PA / IP / appearances, ties); leader-table tie-break by `playerId`; archive round-trip preserves all fields; archive failure aborts cleanly (no partial deletion); checksum mismatch surfaces error without delete; carryover write-back uses sanctioned capability.
- **Integration:** complete Standard season produces a non-empty award doc for every catalog entry; archive + recovery produces identical history-screen output.
- **E2E:** Full Marathon season completes via "Sim full season"; awards screen visual snapshot (mobile + desktop); leaders screen visual snapshot; multi-team manager prompt-routing E2E.
