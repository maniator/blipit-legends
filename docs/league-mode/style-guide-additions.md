# League Mode — Style Guide Additions (proposed)

> Companion to [`README.md`](README.md). **This doc captures the visual tokens and component variants that League Mode v1–v4 needs but `docs/style-guide.md` does not yet ship.** It exists so the league plan is implementable without painting `@ui-visual-snapshot` into a corner. The canonical `docs/style-guide.md` PR is a follow-up owned by `@ux-design-lead` and must merge **before** the v1 implementation PR.

> Resolves UX round-2 P0-3 (five net-new visuals lacked tokens) and P2-15 (`autogen` "Auto" badge token).

## Why this lives in `docs/league-mode/` and not in `docs/style-guide.md`

`docs/style-guide.md` is owned by `@ux-design-lead`. The repo policy is: "before adding any new color, font size, spacing token, or component variant, route to `@ux-design-lead`" (`.github/copilot-instructions.md`). This file is the league-mode PR's **proposal** to `@ux-design-lead`. Once UX signs off, the canonical style guide gains these entries and this file's role narrows to "history of the proposal."

Until then, every UI in League Mode that uses one of these tokens is **blocked on UX sign-off** — captured as a v1 prerequisite in [`agent-prompts/v1.md`](agent-prompts/v1.md).

## Net-new components / variants

### 1. `<StatusPill>` (cross-component primitive)

A small, semantically-tinted pill shown next to player rows, team rows, and library cards.

**Variants required by League Mode:**

| Variant | Used for                                          | Phase |
| ------- | ------------------------------------------------- | ----- |
| `fresh` | Pitcher fatigue 🟢                                | v1    |
| `tired` | Pitcher fatigue 🟡                                | v1    |
| `spent` | Pitcher fatigue 🔴                                | v1    |
| `il`    | Position-player IL status 🚑                      | v2    |
| `auto`  | "Autogenerated team" badge in customTeams library | v1    |
| `team`  | Team name pill in multi-team manager prompts      | v4    |

**Token requirements:**

- Each variant needs a (background, foreground, border) triplet that meets WCAG 2.2 AA contrast against `bgSurface` (`#0d1b2e` per existing palette). Existing fatigue _text_ hex strings (`#ff6b6b` / `#ffd06b` / `#a0b4d0`) need verification — `#ffd06b` on `#0d1b2e` is borderline AA at body sizes per the round-2 a11y review.
- Pill geometry: reuse the existing `theme.radii.pill` plus `NotifBadge`-style padding (extracted from `src/features/gameplay/components/GameControls/styles.ts`).
- Component lives at `src/shared/components/StatusPill/` with named-prop API `variant="…"`.

### 2. `<StatusBanner>`

An inline informational banner — distinct from the existing `UpdateBanner` (which is amber + fixed-position).

**Used for:**

- Roster-edit lock banner inside `CustomTeamEditor` (variant `info`, v1).
- Trade deadline passed banner in Trade Center (variant `warn`, v3).
- Season abandoned banner on historical season pages (variant `neutral`, v1).
- "Continue" affordance on the home screen (variant `info`, v1).

**Token requirements:**

- Three variants: `info`, `warn`, `neutral`. Each pulls from existing semantic color tokens; no new hex.
- Optional `action` slot for a single button/link; disabled-state styling for read-only banners.
- ARIA: `role="status"` for `info`/`neutral`; `role="alert"` for `warn` — to be confirmed by `@ux-design-lead`.

### 3. `<EmptyState>` (icon + title + body + CTA)

Existing `EmptyState` in `docs/style-guide.md §12.2` is a 4-line career-stats variant (text-only, `padding 40px; color #666; 14px`). League Mode wants an enriched pattern with optional icon and call-to-action button.

**Used by:**

- `/leagues` empty hub state (v1)
- `/leagues/:id/transactions` pre-event (v2)
- `/leagues/:id/playoffs` pre-playoff (v3)
- `/leagues/:id/awards` pre-completion (v4)
- RoY award when no rookies eligible (v4)

**Token requirements:**

- Icon slot (24×24 px); existing `Heading` + `Body` + `PrimaryButton` reuse.
- Mobile-first padding scaled via `mq.mobile`/`mq.notMobile`.

### 4. `<BracketConnector>` lines (v3)

Visual lines joining series in `<PlayoffBracketTree>` (desktop only — mobile uses native `<details>` accordion per `playoffs-and-trades.md`).

**Token requirements:**

- Stroke color (likely a muted variant of `borderForm`).
- Stroke weight: 2 px desktop.
- Join treatment: orthogonal joints (no bezier curves).
- No animation in v3 (out of scope).

### 5. `<SortableTable>` extraction

`docs/style-guide.md §10` documents stats-table + line-score-table only. Both are static. League Mode's transactions feed and leaders screen both want column sorting.

**Token requirements:**

- Sort-arrow icon (8×8 px), existing icon library if available; otherwise inline SVG triangle pair.
- Active-sort-direction state (filled vs unfilled triangle).
- Mobile-degradation: collapses to card stack at `${mq.mobile}` (so users on phones get a list-of-cards UI rather than a 2-finger horizontal scroll).
- Keyboard a11y: `aria-sort="ascending" | "descending" | "none"` on column headers; `Enter`/`Space` to toggle.

## Mobile bracket — reuse existing accordion treatment

Per round-2 P2-18: `playoffs-and-trades.md`'s mobile bracket should explicitly inherit `docs/style-guide.md §13` collapsible-section styling (`▸` rotator, `borderForm` outline, etc.), not re-style the `<details>` element. One-line note in `playoffs-and-trades.md`'s "Bracket UI" section captures this — already added in this PR.
