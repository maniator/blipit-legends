# League Mode — Team Autogeneration

> Companion to [`README.md`](README.md). Autogeneration ships in **v1** with all four naming themes and the parity slider. See [`decisions.md`](decisions.md) #A1–#A8 for the locked behaviors.

## Why it exists

Without autogen, a user has to hand-craft 8 / 16 / 24 fully-rostered customTeams before they can start a single season. That is the largest single UX blocker for League Mode. Autogen is the single biggest unlock for v1 and ships with v1.

## One library — autogen writes into `customTeams`

Per [`decisions.md`](decisions.md) #7 and #A3, **autogenerated teams are written into the user's existing `customTeams` collection at setup time.** They appear in the user's library, can be edited via the existing Custom Team Editor, can be exported, can be deleted, and can be reused in future leagues. There is no shadow `leagueTeams` collection and no alternate team type.

Generated teams are tagged with an optional `autogen` marker on their `customTeams` doc:

```
autogen: { version: 1; theme: 'classic'|'scifi'|'whimsical'|'mix'; parity: 'balanced'|'mixed'|'random'; baseSeed: string }
```

The marker is purely informational — it lets us:

- Distinguish "this team was autogenerated" in the Custom Teams library UI (e.g., a small "Auto" badge).
- Re-generate the exact same team for replay debugging if `autogen.version` matches.
- Surface a warning at replay time if `autogen.version` mismatches.

## Modes (selectable in the wizard)

| Mode                | Behavior                                                                     |
| ------------------- | ---------------------------------------------------------------------------- |
| Hand-pick           | No autogeneration; user picks all teams from existing `customTeams`.         |
| **Mixed** (default) | User picks any subset; the wizard autogenerates the rest to fill the preset. |
| All-autogenerated   | One click → entire league of N teams autogenerated. Quickplay path.          |

## Knobs

### Naming theme

All four themes ship in v1. Themes are seeded from the master season seed → reproducible.

| Theme      | Pattern                        | Sample names                               |
| ---------- | ------------------------------ | ------------------------------------------ |
| Classic    | `<city> <animal>`              | "Albany Foxes", "Boise Owls", "Cedar Cubs" |
| Sci-fi     | `<planet/system> <role>`       | "Vega Pilots", "Andromeda Wardens"         |
| Whimsical  | `<food> <verb-er>`             | "Pickle Jugglers", "Waffle Stompers"       |
| Random mix | A random theme picked per team |                                            |

Word lists ship as bundled JSON (~200 first names, ~200 surnames per theme variant, ~50 prefixes). Combined keyspace > 2 million unique team-name combinations per theme; collisions within a single league are explicitly de-duped.

### Stat parity slider

| Setting             | Behavior                                                                      |
| ------------------- | ----------------------------------------------------------------------------- |
| **Balanced**        | All teams within ±5% stat budget. Competitive league with no clear contender. |
| **Mixed** (default) | ±15% spread. Some clear contenders + cellar dwellers, like real MLB.          |
| **Random**          | No balancing — chaos mode.                                                    |

### Roster size

Autogen always produces a roster meeting the **league-play minimum** for the target version (decision #13):

- v1: lineup 9 + bench 3 + 5 SP + 3 RP = **20 active**.
- v2+: lineup 9 + bench 5 + 5 SP + 4 RP = **23 active**.

### Locked seed

Autogen is deterministic per `(masterSeed, autogenSubSeed, teamIndex)`. The wizard's "Re-roll" button changes `autogenSubSeed`, letting the user iterate without committing. The final seed used is captured in each team's `autogen.baseSeed`.

## Generation algorithm

A new pure module: `src/features/league/autogen/`.

Public API (planning shape, exact signatures land with implementation):

```
generateLeagueTeams(options: {
  count: number;
  theme: AutogenTheme;
  parity: AutogenParity;
  masterSeed: string;
  autogenSubSeed: string;
  rosterMinimums: RosterMinimums;
}): CustomTeamShape[]
```

Implementation steps:

1. **Seeded RNG.** Initialize `mulberry32(parseInt(fnv1a(\`${masterSeed}:${autogenSubSeed}\`), 16) >>> 0)`. The `parseInt(..., 16) >>> 0`step converts`fnv1a`'s hex-string output into the uint32 that `mulberry32`requires. All randomness in this module flows through that one PRNG instance — no`Math.random()`.
2. **Stat budget per team.** Allocate a total stat budget; spread across teams per the parity setting (Balanced narrow, Mixed moderate, Random wide).
3. **Position archetypes.** For each team, distribute its budget across positions using fixed archetype weights (catchers favor defense and contact; SS favors defense; corner OF favor power; SP favor stamina/control). Each archetype yields position-appropriate batting/pitching block values within `customTeams` schema constraints.
4. **Names.** Pull from the theme's word lists with the same PRNG; de-dupe within the league. Each player gets first + last name.
5. **Handedness.** Distribute L/R per realistic frequencies (≈25% LH batters, ≈30% LH pitchers).
6. **Pitching role.** Assign exactly the required SP / RP counts, mark eligibility per `pitchingRole` rules ("SP", "RP", "SP/RP" allowed).
7. **Sanitize.** Run the generated team through `sanitizeCustomTeam()` (the existing import path's sanitizer). Any failure is a generator bug, not a user-facing error — emit it via the shared logger.
8. **Tag.** Stamp the `autogen` marker; assign `customTeamId` via `generateTeamId()` and player ids via `generatePlayerId()`.

## Determinism contract

Same `(masterSeed, autogenSubSeed, theme, parity, count, rosterMinimums)` → identical `CustomTeamShape[]` output. This is asserted by unit tests across all four themes and three parity settings.

If the generator is changed in a way that breaks determinism, **bump `autogen.version`**:

- Existing `customTeams` docs keep their old `autogen.version`.
- Replay/seed-sharing UI (v4) compares stored `autogen.version` against current generator version; on mismatch, warns rather than silently regenerating.

## Wizard interactions

- **Re-roll** changes `autogenSubSeed` and re-runs `generateLeagueTeams`. Preview list updates inline.
- **Theme / parity changes** also re-run generation immediately.
- **"Show all generated teams"** expand reveals the team names + their position-by-position summary (mini stat bars).
- **"Save to my teams" is implicit** — generated teams are written into `customTeams` on wizard "Create Season" click. There is no separate save action because there is no separate library.
- **"Discard generated teams"** is _not_ provided in v1; the user can delete them later from the standard Custom Teams page. This avoids partial-write complexity in the wizard.

## Risks & mitigations

| Risk                                                                                    | Mitigation                                                                                                                                                   |
| --------------------------------------------------------------------------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| Autogen rosters feel statistically uniform/boring                                       | Default parity = Mixed (±15%), so leagues have natural strong/weak teams. Post-v1, route a representative season log through `@baseball-manager` for tuning. |
| Naming collisions across many teams (especially Full preset = 24)                       | Generator de-dupes within a single league before returning.                                                                                                  |
| Generator version drift breaks shareable-seed replay later                              | `autogen.version` stored on each generated team. Replay UI surfaces warning on mismatch.                                                                     |
| User edits an autogenerated team in the library, then expects season behavior to change | `seasonTeams.rosterSnapshot` freezes the team at season start, so library edits do not affect in-flight seasons. Setup wizard help text states this.         |
| Library bloat from many seasons of autogen teams                                        | Custom Teams page can filter by `autogen` flag; v4 may add bulk-delete-autogen action. v1 leaves cleanup manual.                                             |

## Testing surface

- **Unit**: `generateLeagueTeams` is pure; cover all themes × parities × counts (8/16/24). Snapshot-test deterministic output.
- **Unit**: name de-duplication within a league (e.g., generate 24 teams, assert no duplicate team name).
- **Unit**: sanitization passes on every generated team across 100 distinct seeds.
- **Component**: autogen panel renders preview list, "Show all" expand works, "Re-roll" updates names.
- **E2E**: setup wizard with All-autogen mode → assert `customTeams` collection grows by exactly N after creation; assert season is created.
