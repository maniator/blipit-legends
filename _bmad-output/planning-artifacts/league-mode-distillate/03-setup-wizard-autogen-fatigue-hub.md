## Setup Wizard

- Entry: Home screen "Start a Season" CTA → `/leagues/new`; Leagues Hub "Create new season" button (disabled w/ tooltip when active season exists)
- Wizard blocks at Step 1 if `seasons` already has an `active` doc → deep-links to existing season (decision #8)
- UI reuse: modal shell from exhibition setup (`max-height: min(90dvh, 820px)` desktop / `min(96dvh, 820px)` mobile; never bare `vh`); `StatusPill` extracted v1; net-new components limited to `LeagueSetupWizard`, `AutogenPanel`, per-step renderers
- Single-page multi-step modal-on-route; all steps backward-reachable; state in local reducer; persisted to `sessionStorage` (survives refresh)
- Step visibility: v1 = Steps 1,2,3,5,6; Step 4 appears v3+

### Step 1 — Preset & length
- League preset radio: Mini · Standard · Full; disabled options show coming-soon badge
- Season length radio: Sprint · Standard · Marathon
- League count: 1–2 in v1; v2+ up to 4; Standard preset = 2 leagues × 2 divisions = 4 division groups
- Validation: ≥1 league

### Step 2 — Teams
- Three modes: Hand-pick (N from customTeams); Mixed (default, subset + autogen fills rest); All autogen (one click)
- Validation: selected + autogen = preset team count; each hand-picked team meets league-play roster minimum
- v1 minimum: lineup 9 + bench 3 + 5 SP + 3 RP = 20 active; v2+: lineup 9 + bench 5 + 5 SP + 4 RP = 23 active (decision #13)
- Teams below minimum → "Auto-fill missing slots" affordance: appends autogen bench/pitchers into season snapshot only, never mutates persistent `customTeams` doc
- Autogen output held in wizard reducer state only until Step 6; no `customTeams` writes at Step 2; abandon → zero orphan docs, no duplicate-key risk on retry

### Step 3 — League rules
- Per-league DH radio: On/Off; quick-presets: Classic AL/NL · Universal DH · Pitchers Hit · Custom; 1-league collapses to single toggle
- Divisions (v2+): per league, None/2/3/4; distribution preview shown
- Series length radio: 3 (default) · 2 · 4

### Step 4 — Trade & playoff rules (v3+)
- Trade deadline: slider 50%–100% of season + "No deadline" checkbox; default 70%
- AI trade aggressiveness: Passive · Moderate (default) · Active
- Playoff format: Default (WC bo3/LCS bo5/BCS bo7) · Short (1/3/5) · Long (5/7/7)

### Step 5 — Seed & advanced
- Master seed: text field pre-populated via `generateFreshSeed()`; Re-roll button
- Advanced (collapsed): "Share seed" toggle (behind `featureFlags.shareableSeasonSeeds`); "Enable minor leagues" (v4+); "Multi-team manager mode" (v4+)

### Step 6 — Review & create (sole persistence point)
- Single transactional create; failure mid-write deletes all partially-created docs including autogen customTeams
- Write sequence: generate IDs → write autogen `customTeams` (idempotent insert) → write `seasons` doc → generate schedule → write `seasonTeams`, `seasonGames`, initial `seasonPlayerState` → navigate to `/leagues/:seasonId`
- Roster-edit lock engages on committed `customTeams` docs (see data-model.md)
- Autogenerated teams become real `customTeams` docs from this point, visible in library (decision #7, #A3)

### State machine
- Reducer-driven, mirrors `editorReducer` in `CustomTeamEditor`
- Key actions: `SET_PRESET`, `SET_SEASON_LENGTH`, `SET_LEAGUE_COUNT`, `SET_TEAM_MODE`, `TOGGLE_TEAM_SELECT`, `SET_AUTOGEN_THEME`, `SET_AUTOGEN_PARITY`, `REROLL_AUTOGEN_SEED`, `AUTOFILL_TEAM_SLOTS`, `SET_DH`, `APPLY_DH_PRESET`, `SET_DIVISION_COUNT`, `SET_SERIES_LENGTH`, `SET_TRADE_DEADLINE`, `SET_TRADE_AGGRESSIVENESS`, `SET_PLAYOFF_FORMAT`, `SET_MASTER_SEED`, `REROLL_MASTER_SEED`, `TOGGLE_FLAG`
- Pure `validateWizardState(state) → string[]`; exhaustively unit-tested

### Accessibility & testing
- All step navigation keyboard-accessible; visible `<label>` per field; errors in `aria-live="polite"`; Re-roll announces new seed
- Unit: reducer + validator (every step, every error path); Component: every step × every mode
- E2E: `e2e/tests/league-setup.spec.ts` — Hand-pick + Mixed + All-autogen → `/leagues/:seasonId`; second E2E: same master seed → identical `seasonGames.derivedSeed[]` arrays

## Team Autogeneration

- Motivation: without autogen, user must hand-craft 8/16/24 rostered teams before any season; autogen is #1 UX unlock for v1
- All four naming themes ship v1; ships with parity slider; locked behaviors in decisions.md #A1–#A8
- Module: `src/features/league/autogen/`

### One-library rule
- Autogenerated teams written into `customTeams` collection at setup time (decision #7, #A3); no shadow `leagueTeams` collection
- Tagged with `autogen: { version, theme, parity, baseSeed }` — informational only; enables "Auto" badge in library; enables replay debug (version match) and mismatch warning (version mismatch)
- Rejected: separate team type / shadow collection

### Naming themes (all v1)
| Theme | Pattern | Sample |
|---|---|---|
| Classic | `<city> <animal>` | "Albany Foxes", "Boise Owls" |
| Sci-fi | `<planet/system> <role>` | "Vega Pilots" |
| Whimsical | `<food> <verb-er>` | "Pickle Jugglers" |
| Random mix | random theme per team | — |
- Word lists: ~200 first names, ~200 surnames per theme variant, ~50 prefixes; keyspace >2M unique combos per theme; de-duped within league

### Parity slider
- Balanced: all teams ±5% stat budget; Mixed (default): ±15% spread; Random: no balancing
- Default = Mixed → natural strong/weak teams like real MLB

### Generation algorithm
1. Seeded RNG: `mulberry32(parseInt(fnv1a(\`${masterSeed}:autogen:${autogenSubSeed}\`), 16) >>> 0)`; `:autogen:` infix namespaces from `:schedule:` and `:tiebreak:` streams; all randomness through one PRNG instance, no `Math.random()`
2. Stat budget per team per parity setting
3. Position archetypes: catchers → defense+contact; SS → defense; corner OF → power; SP → stamina/control; distributes budget within `customTeams` schema constraints
4. Names from theme word lists via PRNG; de-duped within league
5. Handedness: ≈25% LH batters, ≈30% LH pitchers
6. Pitching roles: exactly required SP/RP counts; `pitchingRole` values: `'SP'`, `'RP'`, `'SP/RP'`
7. Sanitize via `sanitizeCustomTeam()` (existing import sanitizer); failure = generator bug → shared logger
8. Tag with `autogen` marker; IDs via `generateTeamId()` / `generatePlayerId()`

### Public API shape
```
generateLeagueTeams(options: {
  count, theme, parity, masterSeed, autogenSubSeed, rosterMinimums
}): CustomTeamShape[]
```

### Determinism contract
- Same `(masterSeed, autogenSubSeed, theme, parity, count, rosterMinimums)` → identical output; unit-tested across all 4 themes × 3 parities
- Generator change breaking determinism → bump `autogen.version`; existing docs keep old version; replay UI warns on mismatch

### Wizard interactions
- Re-roll changes `autogenSubSeed`, re-runs generation, preview updates inline
- Theme/parity changes re-run generation immediately
- "Show all generated teams" expand: team names + position-by-position stat bars
- "Save to my teams" is implicit at Step 6 (no separate save action)
- "Discard generated teams" NOT provided v1; user deletes from Custom Teams page later (avoids partial-write complexity)

### Risks
- Uniform/boring rosters → default parity = Mixed; post-v1 route season log through Buck for tuning
- Naming collisions at Full preset (24 teams) → generator de-dupes within league before returning
- Generator version drift breaks shareable-seed replay → `autogen.version` stored; replay UI warns on mismatch
- User edits autogen team then expects season to change → `seasonTeams.rosterSnapshot` freezes team at season start; library edits don't affect in-flight seasons; wizard help text states this
- Library bloat from many autogen seasons → Custom Teams page filterable by `autogen` flag; v4 may add bulk-delete-autogen; v1 manual cleanup

### Testing
- Unit: `generateLeagueTeams` pure; all themes × parities × counts (8/16/24); snapshot-test deterministic output
- Unit: name de-duplication (generate 24 teams, assert no duplicate team name)
- Unit: sanitization passes across 100 distinct seeds
- Component: autogen panel preview, "Show all" expand, Re-roll updates names
- E2E: All-autogen mode → `customTeams` grows by exactly N after creation; season created

## Fatigue & Injuries

- v1: pitcher arm fatigue; v2: position-player wear + injury system; v3: playoff-mode rules
- Decisions: #6, #9, #10, #11, #12, #19

### v1 — Pitcher Arm Fatigue

#### Data model (fields on `seasonPlayerState`)
- `pitcherDaysRest: number` (0 = pitched today)
- `pitcherAvailability: number` (0..1; 1 = fully fresh)
- `pitcherStartsThisSeason: number` (running counter)

#### State transitions (after every completed `seasonGames`)
- Starting pitcher (both): `pitcherDaysRest = 0`, `pitcherStartsThisSeason++`, recalculate `pitcherAvailability`
- All other pitchers both teams: `pitcherDaysRest++`, `pitcherAvailability` ticks toward 1
- `'SP/RP'` bucketed as `'SP'` for recovery curve (SP outings dominate fatigue accrual)

#### SP recovery curve (~80–110 pitches/outing)
| Days rest | Availability |
|---|---|
| 0 | 0.10 |
| 1 | 0.20 |
| 2 | 0.40 |
| 3 | 0.75 |
| 4 | 1.00 |
| 5+ | 1.00 |
- Eligibility threshold: SP needs `pitcherAvailability ≥ 0.70` to start next game; forbids 1-day-rest starts; 3-day-rest = borderline ("Tired" yellow)

#### RP recovery curve (~10–25 pitches/appearance)
| Days rest | Availability |
|---|---|
| 0 | 0.40 |
| 1 | 0.80 |
| 2 | 0.95 |
| 3+ | 1.00 |
- Eligibility threshold: RP needs `pitcherAvailability ≥ 0.35`; allows realistic 3-of-4-days closer pattern

#### Back-to-back-to-back floor (v1, per Buck + Manager Strategist P3 proxy via Sally)
- RP that appeared in both previous consecutive game-days → `pitcherAvailability` clamped to `0.0` for current game-day regardless of curve
- Prevents AI/manager-mode running a closer 5-of-6 days
- Deterministic, no rolling window; tracked via two existing fields (`pitcherDaysRest === 0` for two prior game-days); no new schema
- Clamp stored in `pitcherFatigueConstants.ts`, pinned to `rulesetVersion`

#### Simulation effect
- `pitcherAvailability` = multiplicative modifier on pitch-quality outcome distribution; max ~12% ("Spent" stats droop ≤12%, decision #11)
- Injected via `seasonModifiers.pitcher.availability`; `SeasonModifiers` shape frozen in v1 (future versions add fields only)

#### Eligibility enforcement
- AI rotation skips ineligible pitchers; rolls to next available SP/RP
- Manager-mode UI grays out ineligible with tooltip "Resting today"

#### v1 AI rotation/lineup policy (blocking before sim PR, per PM-agent dev-readiness review)
- Cycle 5 SPs in `customTeams.pitchers[]` order; skip `pitcherAvailability < 0.70`; if all below threshold, pick highest-availability SP
- Bring highest-availability RP (≥0.35) for late-inning; else no RP change
- Play top-9 lineup as defined in `customTeams.lineup[]` (position-player rest enters v2)
- Quick sim for user's own team: auto-resolves identically to AI policy (same rotation/lineup rules, no extra PRNG calls)

#### UI pill badge
| Availability | Badge | Color |
|---|---|---|
| ≥0.85 | 🟢 Fresh | green |
| 0.50–0.84 | 🟡 Tired | yellow |
| <0.50 | 🔴 Spent | red |
- Shared `<StatusPill variant="fresh|tired|spent|il">` extracted from `src/features/gameplay/components/GameControls/styles.ts` (`NotifBadge` + `theme.radii.pill`); lives at `src/shared/components/StatusPill/`; reused v2 for `🚑 IL` badge
- Rendered on: season team page (rotation card), lineup card during game, manager-mode pitching-change prompts
- Hover/tap tooltip: "X days rest · availability Y%"
- Pill colors must map to existing semantic tokens in `docs/style-guide.md`; raise style-guide PR before new color

### v2 — Position-Player Wear & Injury System

#### Wear
- `wear: number (0..10)` additive field on `seasonPlayerState`
- Starter: `wear += 1` per game; bench: `wear -= 0.5` on rest days, floored at 0
- Effect: max ~5% penalty (decision #11); applied via `seasonModifiers.hitter.wearPenalty`
- Catcher wear multiplier deferred to v2.1 (real catchers ~68% vs other positions ~86%); v2 ships with universal `wear += 1`; flag for Buck retune after Standard season log captured

#### Injury rolls
- Rate: 1.5% per active-lineup player-game regular season (~9 injuries/team/Standard 60-game season); bench + IL players do not roll
- Playoff rate: 0.75% (×0.5, decision #19)
- Constants in `injuryConstants.ts`, pinned to `seasons.rulesetVersion`
- Runs after box-score sim completes (Amelia SC menu review requirement — injury rolls must not perturb at-bat outcomes)
- Iteration order (binding contract for replay): home lineup 1–9, away lineup 1–9, home SP, away SP, home RPs in entrance order, away RPs in entrance order
- Draw uniform[0,1) from per-game `derivedSeed` PRNG for each player
- Injury: sample kind then duration (separate PRNG draws, fixed order)

| Bucket | Probability | Duration (games) |
|---|---|---|
| Short-term | 60% | 1–3 |
| Medium | 30% | 4–10 |
| Long-term | 10% | 11–30 |

- Write `injuryStatus = { kind, ilUntilGameDay }` to `seasonPlayerState`
- Write `seasonTransactions` row: kind `'il_in'`, deterministic id `il:${seasonPlayerStateId}:${gameDay}:il_in`
- Effect: `seasonModifiers.injuryFilter` excludes injured players; AI auto-fills bench; manager-mode prompts user

#### Recovery
- At start of each game-day (before sims): players with `injuryStatus.ilUntilGameDay <= currentGameDay` cleared; write `seasonTransactions` row: kind `'il_out'`, id `il:${seasonPlayerStateId}:${gameDay}:il_out`
- Recovery at start-of-day (not end) is binding contract for replay

#### AI rest behavior (decision #12)
- Position-player rest probability: `restProb = clamp((wear - 6) / 4, 0, 1)`
- Rolled from per-game derived seed after lineup construction, before box-score sim; fixed iteration order (home 1–9, then away 1–9)
- Pitcher rest: strict eligibility threshold (no probability)
- Goal: recognizable lineups (stars play most days) + organic rest cadence without guessable pattern

#### v2 roster minimums (decision #13)
- Bench ≥5; pitchers ≥9 (5 SP + 4 RP = 23 active)
- Buck sign-off: 8 pitchers cannot survive extra-inning or short-start game with full fatigue+injury live
- Setup wizard validates; "auto-fill missing slots from autogen" does not mutate persistent `customTeams` doc

#### v2 UI
- Injury report screen: `/leagues/:seasonId/injuries`
- Transactions feed: `/leagues/:seasonId/transactions`
- Pitcher fatigue badge + "🚑 IL (returns day N)" badge together when injured

### v3 — Playoff Fatigue & Injury Rules (decision #19)
- Pitcher fatigue: full strength (rotation management matters more in playoffs)
- Injury rate: ×0.5 of regular season (0.0075 per active-lineup player-game)
- Position-player wear: unchanged
- Implementation: `playoffMode: boolean` on `seasons` (additive optional field, default `false`, identity migration on new collection); flips multipliers in injury-roll function; no new modifier fields

### Determinism notes
- All injury rolls use per-game `seasonGames.derivedSeed`; same seed = same injuries
- Recovery curves and wear formulas are pure functions of `seasonPlayerState`
- `seasonModifiers` injection: typed structural pass-through; gameplay context never reads RxDB directly
- Injury + wear rolls after box-score sim; recovery clears at start of game-day before sims — keeps fatigue/injury logic changes from perturbing at-bat outcomes for a given seed

### Ruleset versioning (`seasons.rulesetVersion`)
- All tuning constants pinned to `CURRENT_RULESET_VERSION` (integer) from `src/features/league/ruleset/index.ts`
- Every `seasons` doc snapshots `rulesetVersion: number` at create time
- Resume: `season.rulesetVersion === CURRENT_RULESET_VERSION` → normal; else locked to historical ruleset bundled at create-version (constants module exports per-version map)
- Any constant change → must bump `CURRENT_RULESET_VERSION` + add historical entry
- CI guard: `tests/league/rulesetVersionEnforcement.test.ts` snapshots canonical constants per version; diff requires version bump
- Risk without this: every constants tweak silently breaks in-flight saved seasons' replay determinism
- Constants covered: SP/RP recovery curves, eligibility thresholds, max fatigue penalty (12%), wear formula, wear penalty cap (5%), AI rest probability formula, injury rate (1.5% regular / 0.75% playoff), injury duration distribution

### Testing
- Unit (v1): pitcher availability recovery curve; Spent pitchers blocked from starting
- Unit (v2): injury distribution sampler; IL clear-on-recovery; wear floor/ceiling
- Unit (v3): playoff multipliers applied iff `playoffMode === true`
- Integration: 30-game Mini Sprint → believable starts-per-pitcher distribution (no SP starts more than ⌈30/SP_count⌉)
- E2E: same masterSeed → identical injury timelines + identical pitcher-rotation choices

## Leagues Hub (`/leagues`)

### Five canonical hub states
| # | State | Primary action | Secondary action |
|---|---|---|---|
| 1 | Zero seasons, zero customTeams | "Build your first team" → `/teams/new` | "Start a Season" (autogen-only) → `/leagues/new?mode=allAutogen` |
| 2 | Zero seasons, has customTeams | "Start a Season" → `/leagues/new` | "Manage teams" → `/teams` |
| 3 | One active, zero historical | **"Continue: <name> · day 12/30"** → `/leagues/:activeSeasonId` | "View teams" → `/teams` |
| 4 | One active + N historical | **"Continue: <name> · day 12/30"** → `/leagues/:activeSeasonId` | "View past seasons" expands list |
| 5 | Zero active + N historical | "Start a New Season" → `/leagues/new` | "View champion" → `/leagues/:lastSeasonId/awards` (v4) or `/leagues/:lastSeasonId` (v1–v3) |
- "Continue" CTA in states 3/4 is most important affordance for return users; must outrank all other CTAs visually; first focusable element

### Home-screen integration
- No active season → home screen unchanged (Play Ball is primary CTA)
- Has active season → `<StatusBanner variant="info">` at top: "Continue: <name> · day N/M" → `/leagues/:activeSeasonId`; Play Ball remains secondary
- Active league always wins over exhibition save in home-screen "Continue" affordance
- `League teaser box` (style-guide.md §8.6) may be repurposed as container after Sally sign-off in `style-guide-additions.md`

### `/saves` vs `/leagues` split
- `/saves`: single-game exhibition save slots (resume mid-game); nav label updates to **"Exhibitions"** in v1
- `/leagues`: multi-game season records (active + historical); nav label **"Leagues"**
- Home-screen "Continue" points to whichever is more recent (active league always wins)
- Save Curator (P4) proxy interview via Sally owed before v1 ships to validate this split
