## v3 — Trade Engine + Playoff Bracket

- **Goal:** pennant races, deadline drama, playoffs; trade engine + playoff bracket
- **Prerequisites:** v2 shipped + injury system tuned through one full season log review; `seasonTransactions` in production with `kind: string`

## v3 Locked Decisions

- Trade deadline: default = 70% of season; configurable 50%–100% or "no deadline" at setup
- AI trade aggressiveness: Passive 0–1; **Moderate (default)** 1–2; Active 3–4 trades/team/season; 50%+ of trades in last 20% pre-deadline
- User veto on AI ↔ AI trades: none — surfaced in transactions feed only
- Trade scheduler cadence: once per `gameDay`, after start-of-day IL recovery, before any games sim
- Trade scheduler iteration: lex-sorted `seasonTeamId`; one accepted trade per team per cadence; once-moved-cannot-move-again-same-day
- Trade scheduler sub-PRNG: `mulberry32(parseInt(fnv1a(${masterSeed}:trades:day:${gameDay}), 16) >>> 0)`
- Tiebreaker chain: H2H pairwise → intra-division → run-diff vs tied group → run-diff full-season → coin flip from `:tiebreak:${tieKey}` sub-PRNG; iterative for 3-way+
- Playoff format: 4 teams/league; WC bo3 / LCS bo5 / BCS bo7 (default); presets Short (1/3/5), Long (5/7/7)
- Reseeding: after each round, by regular-season record + tiebreaker chain
- Home-field advantage: higher seed hosts game 1, game 2, and any deciding game N; explicit per-series-length host pattern (MLB 2-2-1 / 2-3-2)
- Playoff fatigue: full strength, same recovery + eligibility
- Playoff injury rate: ×0.5 of regular season (decision #19)
- Playoff position-player wear: unchanged
- Bracket persistence: bracket structure persisted to `seasons.bracket` (or `seasonBrackets` collection) at end of regular season
- Bracket UI: vertical accordion (mobile, native `<details>`) / horizontal tree (tablet+desktop) via `mq.mobile`/`mq.notMobile`
- `rulesetVersion`: bump to `3` only if any v1/v2 constant changes; ×0.5 playoff multiplier is v1 value (decision #19) and does not itself require a bump

## v3 Non-Negotiable Contracts

- All v1+v2 contracts hold
- Trade engine determinism: sub-PRNG isolation (`:trades:day:`); lex-sorted iteration; one-accepted-trade-per-team-per-day; once-moved-cannot-move-again-same-day (no A→B→C→A churn); deterministic `proposalId = fnv1a(seasonId + gameDay + sortedTeamPair + sortedPlayerIds)` makes re-running a day no-op
- Trade execution atomicity: update each `seasonPlayerState.seasonTeamId`, update both `seasonTeams.rosterSnapshot`, append `seasonTransactions` row — all under per-game bulk-write contract; rollback on partial failure via RxDB document-version checks
- Tiebreaker PRNG isolation: `mulberry32(parseInt(fnv1a(${masterSeed}:tiebreak:${sortedTiedIds}), 16) >>> 0)`; same `tieKey` always produces same flip
- Three-way+ ties: resolved iteratively (highest seed first → remove → repeat with residual group's `tieKey`)
- Bracket persistence prevents re-derivation drift; resume reads `seasons.bracket`; never re-runs seeding pass
- `TradeProposalDialog` ≠ `DecisionPanel`: no time pressure; side-by-side roster-comparison dialog; borrow modal shell only
- Watch ↔ trade interaction: trade offers arriving during in-flight watch-mode game are queued, surfaced after game completes
- `seasonTransactions.kind` stays `string` (not RxDB enum); trade kinds (`'trade'`, `'trade_aborted'`) validated at app layer
